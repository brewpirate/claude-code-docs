# .claude Directory

Claude Code reads instructions, settings, skills, subagents, and memory from your **project directory** and from `~/.claude` in your home directory. Commit project files to git to share them with your team; files in `~/.claude` are personal and apply across all projects.

Most users only need `CLAUDE.md` and `.claude/settings.json`. Everything else is optional — add skills, rules, or subagents as you need them.

If you set `CLAUDE_CONFIG_DIR`, every `~/.claude` path on this page lives under that directory instead.

---

## Project files

Files that live in your project root and `.claude/` folder. Most should be committed so your whole team shares them.

<FileTree>

- your-project/
  - **CLAUDE.md** Project instructions loaded into context every session
  - .mcp.json Team-shared MCP server configuration
  - .worktreeinclude Gitignored files to copy into new worktrees
  - **.claude/**
    - **settings.json** Permissions, hooks, model, env vars
    - settings.local.json Personal overrides — gitignored automatically
    - rules/
      - testing.md Path-gated: loads when Claude reads a matching test file
      - api-design.md Path-gated: loads when Claude reads src/api/
    - skills/
      - security-review/
        - **SKILL.md** Entrypoint: trigger, invocability, instructions
        - checklist.md Supporting file bundled with the skill
    - commands/
      - fix-issue.md Invoked as /fix-issue — prefer skills/ for new work
    - output-styles/ Project-scoped output style overrides
    - agents/
      - code-reviewer.md Subagent with its own prompt, tools, and model
    - agent-memory/
      - code-reviewer/
        - MEMORY.md Claude writes and maintains this automatically

</FileTree>

| File | When it loads | Reference |
|------|--------------|-----------|
| `CLAUDE.md` | Every session, alongside `~/.claude/CLAUDE.md` | [Memory](/claude-code-docs/memory/overview/) |
| `.mcp.json` | Session start — connects team MCP servers | [Settings → MCP Servers](/claude-code-docs/settings/mcp-servers/) |
| `.worktreeinclude` | When Claude creates a git worktree | [Sessions](/claude-code-docs/sessions/overview/) |
| `.claude/settings.json` | Overrides `~/.claude/settings.json` | [Settings](/claude-code-docs/settings/overview/) |
| `.claude/settings.local.json` | Highest user-editable priority (gitignored) | [Settings](/claude-code-docs/settings/overview/) |
| `.claude/rules/*.md` | Session start (no `paths:`) or when a matching file enters context (`paths:`) | [Memory](/claude-code-docs/memory/overview/) |
| `.claude/skills/<name>/SKILL.md` | On `/name` invocation or Claude auto-match | [Skills](/claude-code-docs/skills/overview/) |

## Global files (`~/.claude`)

Files in your home directory. These apply to every project and are never committed.

<FileTree>

- ~/
  - .claude.json App state, UI toggles, OAuth session, personal MCP servers
  - **.claude/**
    - **CLAUDE.md** Personal preferences across all projects
    - **settings.json** Default settings for all projects
    - keybindings.json Custom keyboard shortcuts (hot-reloaded on edit)
    - projects/ Auto memory — Claude writes and reads these
      - project-name/memory/
        - MEMORY.md Index loaded at session start (first 200 lines / 25 KB)
        - debugging.md Topic file — read on demand when related task comes up
    - rules/ User-level rules that apply everywhere
    - skills/ Personal skills available in every project
    - commands/ Personal single-file commands (all projects, legacy)
    - output-styles/ Personal output styles (all projects)
    - agents/ Personal subagents available in every project
    - agent-memory/ User-scoped subagent persistent memory

</FileTree>

## Full file reference

| File | Scope | Committed | What it does | Reference |
|------|-------|-----------|--------------|-----------|
| `CLAUDE.md` | Project + global | ✓ | Instructions loaded every session | [Memory](/claude-code-docs/memory/overview/) |
| `rules/*.md` | Project + global | ✓ | Topic-scoped instructions, optionally path-gated | [Memory](/claude-code-docs/memory/overview/) |
| `settings.json` | Project + global | ✓ | Permissions, hooks, env vars, model defaults | [Settings](/claude-code-docs/settings/overview/) |
| `settings.local.json` | Project only | gitignored | Personal overrides, auto-gitignored | [Settings](/claude-code-docs/settings/overview/) |
| `.mcp.json` | Project only | ✓ | Team-shared MCP servers | [Settings → MCP Servers](/claude-code-docs/settings/mcp-servers/) |
| `.worktreeinclude` | Project only | ✓ | Gitignored files to copy into new worktrees | [Sessions](/claude-code-docs/sessions/overview/) |
| `skills/<name>/SKILL.md` | Project + global | ✓ | Reusable prompts invoked with `/name` or auto-invoked | [Skills](/claude-code-docs/skills/overview/) |
| `commands/*.md` | Project + global | ✓ | Single-file prompts; same mechanism as skills (legacy) | [Skills](/claude-code-docs/skills/adding-your-own-skills/) |
| `output-styles/*.md` | Project + global | ✓ | Custom system-prompt sections | [Settings](/claude-code-docs/settings/overview/) |
| `agents/*.md` | Project + global | ✓ | Subagent definitions with their own prompt and tools | [Agents](/claude-code-docs/agents/overview/) |
| `agent-memory/<name>/` | Project + global | ✓ | Persistent memory for subagents | [Agents](/claude-code-docs/agents/how-subagents-work/) |
| `~/.claude.json` | Global only | local | App state, OAuth, UI toggles, personal MCP servers | [Settings](/claude-code-docs/settings/overview/) |
| `~/.claude/projects/*/memory/` | Global only | local | Auto memory: Claude's notes across sessions | [Memory → Auto Memory](/claude-code-docs/memory/auto-memory/) |
| `~/.claude/keybindings.json` | Global only | local | Custom keyboard shortcuts | [Keybindings](/claude-code-docs/keybindings/overview/) |

:::note[Settings precedence]
Managed settings deployed by your organization take precedence over everything, CLI flags like `--permission-mode` override `settings.json` for that session, and some environment variables override their equivalent setting. See the [Settings overview](/claude-code-docs/settings/overview/) for the full precedence order.
:::

## Check what loaded

Run these slash commands to inspect what is active in the current session.

| Command | Shows |
|---------|-------|
| `/context` | Token usage by category: system prompt, memory files, skills, MCP tools, messages |
| `/memory` | Which `CLAUDE.md` and rules files loaded, plus auto-memory entries |
| `/agents` | Configured subagents and their settings |
| `/hooks` | Active hook configurations |
| `/mcp` | Connected MCP servers and their status |
| `/skills` | Available skills from project, user, and plugin sources |
| `/permissions` | Current allow and deny rules |
| `/doctor` | Installation and configuration diagnostics |

Start with `/context` for an overview, then use the specific command for the area you want to investigate.

## Application data

Beyond config you author, `~/.claude` holds data Claude Code writes during sessions. These files are plaintext — anything that passes through a tool (file contents, command output, pasted text) lands in a transcript on disk.

### Cleaned up automatically

Deleted on startup once older than [`cleanupPeriodDays`](/claude-code-docs/settings/overview/) (default: 30 days).

| Path under `~/.claude/` | Contents |
|-------------------------|---------|
| `projects/<project>/<session>.jsonl` | Full transcript: every message, tool call, and result |
| `projects/<project>/<session>/tool-results/` | Large tool outputs spilled to separate files |
| `file-history/<session>/` | Pre-edit snapshots used for checkpoint restore |
| `plans/` | Plan files written during plan mode |
| `debug/` | Per-session debug logs (only with `--debug` or `/debug`) |
| `paste-cache/`, `image-cache/` | Contents of large pastes and attached images |
| `session-env/` | Per-session environment metadata |

### Kept until you delete them

| Path under `~/.claude/` | Contents |
|-------------------------|---------|
| `history.jsonl` | Every prompt you've typed, with timestamp and project path |
| `stats-cache.json` | Aggregated token and cost counts shown by `/cost` |
| `backups/` | Timestamped copies of `~/.claude.json` from config migrations |

:::caution[Plaintext storage]
Transcripts and history are not encrypted at rest. If a tool reads a `.env` file or a command prints a credential, that value lands in `projects/<project>/<session>.jsonl`. Lower `cleanupPeriodDays` to reduce retention, or set `CLAUDE_CODE_SKIP_PROMPT_HISTORY` to skip writing transcripts entirely.
:::

Do not delete `~/.claude.json`, `~/.claude/settings.json`, or `~/.claude/plugins/` — those hold your auth, preferences, and installed plugins.
